# Self-Editing Memory Design

## Concept
The agent can autonomously write to its own Brain memory — not just when asked, but when it recognizes learnable moments. This makes the agent self-improving over time.

## Architecture

### Memory Write Triggers
The agent writes to memory when:
1. **Explicitly asked**: "Remember this..." / "Note that..."
2. **Implicitly useful**: After a complex debugging session, store the solution pattern
3. **Error patterns**: After recovering from an error, store what went wrong + fix
4. **User preferences**: After observing user preferences over multiple interactions
5. **Task completion**: After a significant task, store what was done + how

### Memory Types
```typescript
interface MemoryEntry {
  id: string;
  type: 'explicit' | 'implicit' | 'error_pattern' | 'preference' | 'task_summary';
  key: string;
  value: string;
  confidence: number;      // 0-1, how sure we are this is useful
  accessCount: number;     // how often this has been retrieved
  lastAccessed: string;
  createdAt: string;
  expiresAt?: string;      // optional TTL for low-confidence memories
  source: string;          // which task/session created this
}
```

### Budget System
Prevent memory bloat:
- Max memory writes per session: 10 (configurable)
- Max total memories: 1000 (configurable)
- Confidence threshold for auto-write: 0.7
- Low-confidence memories decay: if not accessed in 30 days, confidence -= 0.1
- Below 0.3 confidence: eligible for deletion

### Forgetting
```typescript
class MemoryManager {
  // Periodic cleanup (called on heartbeat)
  async prune(): Promise<{ deleted: number; decayed: number }>;
  
  // Decay confidence of unused memories
  async decay(): Promise<number>;
  
  // Get memories worth keeping
  async getValuable(threshold?: number): Promise<MemoryEntry[]>;
}
```

### Integration with Brain
Extend Brain's setFact to accept metadata:
```typescript
brain.setFact(key, value, {
  type: 'implicit',
  confidence: 0.8,
  source: 'task-123',
  autoGenerated: true
});
```

### Safety
- Auto-written memories are tagged `autoGenerated: true`
- User can list and delete auto-written memories
- Explicit memories (user-requested) never expire
- Auto-written memories go through a lightweight validation:
  - Not a duplicate of existing memory
  - Not contradictory to explicit memories
  - Not PII (run PII detector before storing)

### Token Impact
- Average memory: ~50 tokens
- 100 useful memories: ~5000 tokens
- Only inject top-5 most relevant memories per request (based on hybrid search)
- Net effect: +250 tokens per request for 5x better context
